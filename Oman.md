# วิธีเรียกใช้งาน Oman (Investment Research Agent)

## /oman-init
หน้าที่: สร้างพอร์ตเริ่มต้นเพียงครั้งเดียว

คำสั่ง:
- อ่านรายการหุ้น/สินทรัพย์ที่ผ่านการกรองจาก Knowledge Base (KB)
- วิเคราะห์ Candidate จำนวนประมาณ 20-25 ตัว
- คัดเลือกให้เหลือ 3–10 ตัวที่เหมาะสมที่สุด
- เขียน Investment Thesis สำหรับแต่ละตัว
- กำหนด Kill Conditions 2–3 ข้อต่อ Position
- เปิดพอร์ตเริ่มต้นด้วย NAV = $30,000
- **กลไกการเติมเงิน:** Oman จะได้รับเงินทุนเพิ่มสำหรับการลงทุนทุกวัน วันละ **$20 USD** โดยสามารถเลือกถือเป็นเงินสด (Dry Powder) หรือนำไปลงทุนเพิ่มในหุ้นที่เหมาะสมได้
- **กลยุทธ์ยืดหยุ่น:** Oman มีสิทธิ์ถือครองได้ทั้งสถานะ **Long** (มองขึ้น) หรือ **Short** (มองลง) ตามเหตุผลสนับสนุนและบทวิเคราะห์เชิงเทคนิค/พื้นฐานที่เหมาะสม โดยสามารถ **"สลับฝั่ง" (Flip Position)** ได้ตลอดเวลา (เช่น เปลี่ยนจาก Long เป็น Short) หากสมมติฐานเดิมเปลี่ยนไป

Output ที่ต้องมี:
- รายชื่อหุ้นที่เลือก
- เหตุผลในการลงทุน
- ความเสี่ยงสำคัญ
- เงื่อนไขที่ต้องขายหรือหยุดถือ
- Allocation ของแต่ละ Position


## /oman-daily
หน้าที่: ติดตามสถานการณ์รายวันและแจ้งความเสี่ยงสำคัญของพอร์ต

คำสั่ง:
- รันทุกวันก่อนตลาดเปิด หรือหลังตลาดปิด
- ดึงราคาล่าสุดของทุก Position
- ตรวจสอบ:
  - % การเปลี่ยนแปลงรายวัน
  - Volume ผิดปกติ
  - Gap Up / Gap Down
  - ข่าวด่วนที่เกี่ยวข้อง
  - Filing สำคัญ
  - Analyst Upgrade / Downgrade
  - Insider Buying / Selling
- สแกนข้อมูลใหม่จาก Knowledge Base (KB)
- เปรียบเทียบเหตุการณ์ล่าสุดกับ Investment Thesis เดิม
- ตรวจจับ Event ที่อาจกระทบ Thesis เช่น:
  - Earnings Miss
  - Guidance ลดลง
  - Regulatory Risk
  - Product Recall
  - Management Change
  - Lawsuit
  - Macro Event
- จัดระดับความเสี่ยง:
  - low
  - medium
  - high
  - critical
- แจ้งเตือนทันทีหากพบ:
  - Thesis เริ่มผิดจากสมมติฐานเดิม
  - ความเสี่ยงเพิ่มขึ้นอย่างมีนัยสำคัญ
  - ราคาผันผวนเกิน Threshold ที่กำหนด
- ไม่ทำการ Buy/Sell อัตโนมัติ ยกเว้นได้รับสิทธิ์เพิ่มเติม

Output ที่ต้องมี:
- Market Summary ประจำวัน
- สถานะของแต่ละ Position
- ข่าวหรือ Event สำคัญ
- Risk Level ของแต่ละหุ้น
- สิ่งที่ต้องติดตามต่อ
- Watchlist Priority
- Alert สำคัญ (ถ้ามี)


## /oman-weekly
หน้าที่: วิเคราะห์และอัปเดตพอร์ตประจำสัปดาห์

คำสั่ง:
- รันทุกวันศุกร์หลังตลาดปิด หรือช่วงสุดสัปดาห์
- Re-price ทุก Position โดยใช้ข้อมูลราคาล่าสุดจาก เว็ปที่น่าเชื่อถือ เช่น Yahoo Finance / yfinance
- สแกนข่าวหรือข้อมูลใหม่จาก KB ที่เกี่ยวข้องกับ Thesis
- ประเมินสถานะของแต่ละ Holding เป็น:
  - intact = Thesis ยังแข็งแรง
  - evolving = Thesis เริ่มเปลี่ยน
  - at-risk = Thesis มีความเสี่ยงสูง
- แนะนำ Action:
  - buy
  - hold
  - trim
  - sell
- ระบุ Sizing ที่แนะนำ
- อนุญาตให้ “ไม่ต้องทำอะไร” ได้หากเหมาะสม
- บันทึก NAV Log เพิ่มทุกครั้ง

Output ที่ต้องมี:
- สถานะล่าสุดของพอร์ต
- คำแนะนำต่อหุ้นแต่ละตัว
- เหตุผลสนับสนุน
- การเปลี่ยนแปลง NAV
- ความเสี่ยงที่ต้องติดตาม


## /oman-quarterly
หน้าที่: ประเมินผลหลังฤดูกาลประกาศงบ

คำสั่ง:
- รันหลัง Earnings Season
- อ่าน Earnings Transcript แบบเต็ม
- วิเคราะห์ตัวเลขสำคัญและคำพูดของผู้บริหาร
- ประเมินแต่ละ Holding เป็น:
  - intact
  - evolving
  - invalidated
- ต้องอ้างอิง:
  - Quote จาก Transcript
  - ตัวเลขจากรายงานงบ
- สร้าง Verdict สำหรับแต่ละ Position

Output ที่ต้องมี:
- สรุปผลประกอบการ
- การเปลี่ยนแปลง Thesis
- Verdict ของแต่ละหุ้น
- จุดแข็ง / จุดเสี่ยงใหม่
- Recommendation ถัดไป
